# Как запустить сайт — инструкция на пальцах

Тебе нужен только **Docker Desktop** (у тебя уже есть).

## Шаг 0. Убедись что Docker запущен

Найди в трее/меню иконку Docker (кит). Она должна быть **зелёной / running**.
Если нет — открой Docker Desktop и подожди 20–30 секунд, пока он полностью поднимется.

---

## Шаг 1. Запустить сайт

### Windows

Просто **двойной клик на `start.bat`** в папке проекта. Всё.

Если двойной клик не сработал — ПКМ → «Запустить от имени администратора».

### macOS / Linux

Открой Terminal, перейди в папку проекта и выполни:
```bash
./start.sh
```
(Если пишет «permission denied» — сначала `chmod +x start.sh`, потом снова.)

### Ручной способ (если хочешь понимать что происходит)

```bash
cd путь/к/папке/ICT
docker compose up --build
```

Первый раз будет долго (2–5 минут) — Docker качает образы Python и Postgres, собирает твой образ. Следующие запуски — 5–10 секунд.

---

## Шаг 2. Открыть в браузере

```
http://localhost:5000
```

Скрипт открывает сам, но можешь вбить вручную.

Страницы сайта:
- `/` — главная
- `/passwords` — генератор, сила, проверка утечек
- `/encryption` — шифрование/расшифровка
- `/about` — о проекте

---

## Остановить сайт

В том же терминале нажми `Ctrl+C`. Или в новой команде:
```bash
docker compose down
```

Данные Postgres при этом сохраняются в volume. Если хочешь удалить ещё и данные:
```bash
docker compose down -v
```

---

## Если что-то не работает

**«docker compose: command not found»**
→ Обнови Docker Desktop до свежей версии (в старых была `docker-compose` через дефис — тоже подойдёт).

**«port is already allocated» / «5000 занят»**
→ У тебя уже что-то на 5000. В `docker-compose.yml` поменяй строку
`"5000:5000"` на `"5001:5000"` — и открывай `http://localhost:5001`.

**Белый экран / 500 ошибка**
→ Смотри логи:
```bash
docker compose logs -f web
```

**Сайт не открывается**
→ Подожди 10 секунд после запуска — контейнеры поднимаются не мгновенно, Postgres проходит healthcheck.

## Что потом

Когда локально всё работает:
1. Запушить репозиторий на GitHub.
2. Создать Web Service на [Render](https://render.com) → подключить репо → выбрать Docker.
3. Добавить PostgreSQL (Render → New → PostgreSQL).
4. Пробросить переменные окружения из `.env`.
5. Получить публичный URL вида `https://dsp-xyz.onrender.com` с HTTPS из коробки.

Всё это подробно в `README.md` и `docs/report.md`.
